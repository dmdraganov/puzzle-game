
# Документация по проекту "Игра-пазл"

## Обзор

Это веб-приложение представляет собой интерактивную игру-пазл. Цель игры - собрать полную картинку, перетаскивая ее отдельные фрагменты на правильные места на игровом поле.

## Архитектура

Приложение построено с четким разделением ответственности между логикой игры и ее визуальным представлением.

### Модульная структура

-   **`main.js`**: Точка входа в приложение. Инициализирует игру и представление.
-   **`game/`**: Содержит всю игровую логику.
    -   `game.js`: Ядро игры. Управляет состоянием (положением фигур, сеткой, очками, таймером), обрабатывает ходы и проверяет условия победы. Не имеет прямого доступа к DOM.
    -   `piece.js`: Класс, представляющий отдельный элемент пазла, его свойства и состояние.
    -   `edges.js`: (Потенциально) Логика для определения правильности соединения соседних элементов.
-   **`view/`**: Отвечает за все, что видит и с чем взаимодействует пользователь.
    -   `view.js`: Управляет отображением игрового поля и элементов, обрабатывает пользовательский ввод (касания, перетаскивание), запускает анимации и визуальные эффекты. Вызывает методы игрового модуля (`game.js`) для обновления состояния игры.
-   **`utils/`**: Вспомогательные модули.
    -   `storage.js`: Функции для сохранения и загрузки прогресса игры (используя `localStorage`).
    -   `timer.js`: Управляет игровым таймером.
    -   `utils.js`: Общие вспомогательные функции.

### Взаимодействие модулей

`view.js` выступает в роли контроллера, который реагирует на действия пользователя и вызывает соответствующие методы в `game.js` (валидатор).

1.  Пользователь перетаскивает элемент пазла. `view.js` самостоятельно отслеживает все промежуточные координаты.
2.  Когда пользователь отпускает пазл, `view.js` определяет, над какой ячейкой сетки это произошло.
3.  `view.js` вызывает метод `game.isCorrectPosition(pieceId, gridX, gridY)`, чтобы проверить, является ли эта ячейка правильной для данного пазла.
4.  `game.js` возвращает `true` или `false`.
5.  На основе ответа, `view.js` обновляет DOM: либо фиксирует пазл как правильный (с анимацией), либо просто оставляет его в этой ячейке сетки.

## Основные функции

1.  **Игровое поле и элементы**: Динамически создаваемое поле (сетка) и элементы пазла.
2.  **Перемещение**: Элементы можно свободно перетаскивать.
3.  **Притягивание (Snapping)**: Элемент автоматически "прилипает" к правильной ячейке сетки, если он достаточно близко.
4.  **Анимация соединения**: Плавная анимация при правильном размещении.
5.  **Мультитач**: Возможность перемещать несколько элементов одновременно.
6.  **Счет и время**: Система подсчета очков и таймер.
7.  **Эффекты завершения**: Визуальные эффекты при успешной сборке пазла.
8.  **Сохранение прогресса**: Автоматическое сохранение состояния игры.
9.  **Адаптивность**: Интерфейс адаптируется под разные размеры экрана.
10. **Тактильная отдача**: Вибрация при успешном соединении элемента (на поддерживаемых устройствах).
